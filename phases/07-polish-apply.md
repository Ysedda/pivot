# Phase 7 — Polish + apply

**Duration:** 2 weeks  
**Target end:** 2026-12-01  
**Hours budget:** ~20

## Goal

Turn the work into a portfolio that closes loops in interviewers' heads on its own, and start applying.

## Why this phase

Most candidates have the experience but lose interviews on signal: messy READMEs, no diagrams, no narrative. This phase is unglamorous but high-ROI.

## Deliverable

A portfolio that someone can skim in 10 minutes and know:

- What problem each repo solves.
- The architecture (a real diagram, not boxes-and-arrows on a napkin).
- What you learned, what you'd do differently.
- A "want to verify this is real?" path: how to bring it up locally.

## Checklist

### READMEs (every repo / sub-directory)
- [ ] One sentence at the top: what this is.
- [ ] One paragraph: why it exists, what problem it solves.
- [ ] Architecture diagram (Mermaid in the README is fine; for complex ones, an image checked in).
- [ ] How to run it locally (or "this is lab-only, here's the topology").
- [ ] What you'd do differently — the most underrated section.

### Blog posts (2–3, deep)
- [ ] Pick topics from real surprises in `tasks/lessons.md`. Suggested:
  - "Walking Kubernetes the Hard Way on a Pi + VPS — what actually clicked"
  - "SLOs for a side project: what's worth measuring when you have one user"
  - "Going from click-ops to Terraform + Argo CD: the parts the tutorials skip"
- [ ] Publish on a personal site, dev.to, or Medium. Cross-link from this repo.

### CV / LinkedIn
- [ ] One-pager CV optimized for SRE/Platform roles. Lead with concrete deliverables, link to repos.
- [ ] LinkedIn headline rewrite. Headline = role you want, not role you have.

### Apply
- [ ] Refine target company list to 10–15 from phase 6.
- [ ] First applications go out before this phase ends.
- [ ] For each application, customize the CV + cover letter to one specific deliverable in this repo that maps to their stack.

## Resources

- *The Tech Resume Inside Out* (Orosz) — reference for SRE-flavored CVs.
- "How to write a great README" — too many guides; pick one and stop bikeshedding.
- Mermaid live editor — for diagrams that live in markdown.

## Stretch

- Record a 5-minute Loom walkthrough of the lab. Uncomfortable but very effective in cover-letter links.
- Open-source one piece of the capstone (operator, CLI, dashboard pack). Even with zero stars it shows ability to ship public.

## Lessons

Append § Phase 7 to `tasks/lessons.md` — final entry, including: where the program over-delivered, where it under-delivered, what you'd cut or expand if doing it again.
